Keyboard key mapping (and window manager)

Fedora comes with a Gnome version that uses Wayland as a window manager. However, it is very hard to remap keys with Wayland.

Thus, first, switch to X.org:

/etc/gdm/custom.conf

# GDM configuration storage

[daemon]
# Uncoment the line below to force the login screen to use Xorg
WaylandEnable=false
DefaultSession=gnome-xorg.desktop


[security]

[xdmcp]

[chooser]

[debug]
# Uncomment the line below to turn on debugging
#Enable=true

(ref: https://ask.fedoraproject.org/en/question/96134/how-can-i-set-xorg-as-a-default-on-fedora-25/)

Apple Keyboard

~/.Xmodmap_appleKeyboard

! all keys: Alt_L Alt_R Meta_L Meta_R Super_L Super_R Control_L Control_R Hyper_L Hyper_R

! pyhsical keys:
! 108: Alt_R
! 134: Cmd_R
!  64: Alt_L
! 133: Cmd_L
! 204: don't know, but assigned to Alt_L as another key by default
! 205: don't know, but assigned to Meta_L by default
! 206: don't know, but assigned to Super_L by default
! 207: don't know, but assigned to Hyper_L by default

! relevant mods:
! mod1: alt
! mod4: super

! clear relevant mods
clear mod1
clear mod4

! clear assignments for physical keys that I don't know
! keycode 204 = NoSymbol
! keycode 205 = NoSymbol
! keycode 206 = NoSymbol
! keycode 207 = NoSymbol

! make sure keycode 108 (pyhsical Alt_R key) is set to Meta_R and Alt_R
keycode 108 = Meta_R
! make sure keycode 134 (pyhsical Cmd_R key) is set to Super_R
keycode 134 = Super_R

! -------------------------

! remove Meta_L assignment from physical Alt_L key.
keycode 64 = Alt_L
		
! assign physical Cmd_L key to Meta_L
keycode 133 = Meta_L


! -------------------------

! make Alt_L the Alt mod
add mod1 = Alt_L
! make Cmd_L the Super mod
add mod4 = Meta_L

Apple Keyboard fn-key fix

See:

  • https://www.dalemacartney.com/2013/06/14/changing-the-default-function-key-behaviour-in-fedora/
  • https://superuser.com/questions/79822/how-to-swap-the-fn-use-of-function-keys-on-an-apple-keyboard-in-linux

Doing echo 2 > /sys/module/hid_apple/parameters/fnmode with sudo would work.

However, to persist the setting, let’s create a service that is enabled upon boot.

Create /etc/systemd/system/apple-keyboard.service with this content:

[root@localhost system]# cat /etc/systemd/system/apple-keyboard.service 
[Unit]
 Description=apple-keyboard
[Service]
 Type=oneshot
 ExecStart=/bin/sh -c "echo 2 > /sys/module/hid_apple/parameters/fnmode"
 ExecStop=/bin/sh -c "echo 1 > /sys/module/hid_apple/parameters/fnmode"
 RemainAfterExit=yes
[Install]
 WantedBy=multi-user.target
[root@localhost system]#

Enable it:

chcon -t systemd_unit_file_t  /etc/systemd/system/apple-keyboard.service
systemctl daemon-reload
